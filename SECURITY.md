# Security Policy

## 支持范围

当前支持 `main` 分支上的最新发布内容，以及正在 `develop` 分支准备中的 CLI 改动。

## 报告安全问题

如果你发现与本仓库相关的安全问题，请不要在公开 issue 中贴出可利用细节、token、API key、密码、私钥或个人敏感信息。

建议先通过 GitHub 私下联系维护者，或创建一个不包含敏感细节的 issue，说明需要私下沟通安全问题。

适合报告的问题包括：

- 安装、更新或移除插件时意外覆盖用户配置。
- `hooks.json` 或 `mcp.json` 合并逻辑存在安全风险。
- 远程 marketplace 或 GitHub API 下载逻辑指向错误或不可信内容。
- CLI 输出、文档或示例可能诱导用户泄露凭据。

## 安装者安全提示

安装来自未知 marketplace 的插件前，请先确认仓库来源可信。安装包含 hooks 的插件时，CLI 会提示风险；请审查实际内容后再继续。

不要盲目信任未知 hook，也不要把 API key、token 或机器专属路径写进插件配置或输出。
